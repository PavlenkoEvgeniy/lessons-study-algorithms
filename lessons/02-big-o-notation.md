# Урок 2: Как измерить скорость алгоритма (Big O)

## Зачем этот урок

Представь: ты выбираешь маршрут до работы.
- Прямой путь: 30 минут
- Через магазин: 10 минут пешком + 5 минут в магазине + 20 минут = 35 минут

Понятно, что прямой путь быстрее. **Алгоритмы работают также** — у каждого есть "время выполнения".

---

## Что такое Big O?

**Big O** — это способ сказать "насколько медленным станет алгоритм, когда данных станет много".

Формально: описывает, как растёт время выполнения при росте входных данных.

Неформально: "грубо говоря, сколько времени" — т.е. насколько медленным станет алгоритм.

---

## Четыре главных скорости

### O(1) — Постоянное время
**Значение:** Время НЕ зависит от количества данных.

```
Пример: посмотреть первый элемент списка
Неважно 10 элементов или 10 миллионов — время одно и то же
```

```php
<?php

// O(1) — мгновенно, не зависит от размера
function getFirst(array $arr)
{
    return $arr[0]; // Просто взяли первый элемент
}

// O(1) — проверить чётность числа
function isEven(int $n): bool
{
    return $n % 2 === 0;
}

$arr = [1, 2, 3, 4, 5];
echo getFirst($arr); // Всегда одно действие
```

**Визуализация:**
```
Данные:     10        100       1000      10000
Время:      ████       ████       ████       ████  (всегда одно и то же)
```

---

### O(n) — Линейное время
**Значение:** Во сколько раз больше данных — во столько же раз больше время.

```
Пример: найти друга в списке
10 друзей = проверить 10 раз
100 друзей = проверить 100 раз
```

```php
<?php

// O(n) — линейный поиск
function findFriend(array $friends, string $name): bool
{
    foreach ($friends as $friend) {
        if ($friend === $name) {
            return true;
        }
    }
    return false;
}

// O(n) — найти сумму
function sumArray(array $numbers): int
{
    $sum = 0;
    foreach ($numbers as $num) {
        $sum += $num;
    }
    return $sum;
}
```

**Визуализация:**
```
Данные:     10        100       1000       10000
Время:      ████       ██████████████       ██████████████████████████
           (быстро)      (медленнее)            (ещё медленнее)
```

---

### O(log n) — Логарифмическое время
**Значение:** При удвоении данных — время растёт на ОДНУ операцию.

```
Это как поиск слова в словаре:
- Открыли случайную страницу
- Поняли, что слово левее или правее
- Открыли нужную половину
- Повторили

10 страниц = 4 проверки
1000 страниц = 10 проверок
1000000 страниц = 20 проверок
```

```php
<?php

// O(log n) — бинарный поиск
// Работает ТОЛЬКО с отсортированным массивом!

function binarySearch(array $sorted, int $target): bool
{
    $left = 0;
    $right = count($sorted) - 1;
    
    while ($left <= $right) {
        $mid = (int)(($left + $right) / 2);
        
        if ($sorted[$mid] === $target) {
            return true; // Нашли!
        } elseif ($sorted[$mid] < $target) {
            $left = $mid + 1; // Ищем справа
        } else {
            $right = $mid - 1; // Ищем слева
        }
    }
    
    return false;
}

// Пример: ищем 7 в отсортированном массиве
$sorted = [1, 3, 5, 7, 9, 11, 13];

// Шаг 1: left=0, right=6, mid=3, значение=7
//         7 == 7? ДА! Нашли!
// Вместо 7 проверок линейного — только 1!
```

**Визуализация:**
```
Данные:        10        100        1000        10000
Время:         ████       ██████       ████████       ██████████
            (едва растёт)   (медленно растёт)    (совсем медленно)
```

**Сравнение O(n) vs O(log n):**
```
n=1,000,000 элементов:

Линейный поиск O(n):    проверим 1,000,000 раз
Бинарный поиск O(log n): проверим всего 20 раз!
                            (log₂(1,000,000) ≈ 20)
```

### O(n²) — Квадратичное время
**Значение:** При удвоении данных — время увеличивается в 4 раза!

```
Пример: сравнить каждого с каждым
10 человек = 100 сравнений (10 × 10)
100 человек = 10,000 сравнений (100 × 100)
```

```php
<?php

// O(n²) — пузырьковая сортировка
function bubbleSort(array &$arr): void
{
    $n = count($arr);
    
    // Внешний цикл: n раз
    for ($i = 0; $i < $n; $i++) {
        
        // Внутренний цикл: тоже n раз
        for ($j = 0; $j < $n; $j++) {
            // Сравниваем соседей
            if ($arr[$j] > $arr[$j + 1]) {
                $temp = $arr[$j];
                $arr[$j] = $arr[$j + 1];
                $arr[$j + 1] = $temp;
            }
        }
    }
}

// O(n²) — проверить дубликаты
function hasDuplicates(array $arr): bool
{
    $n = count($arr);
    
    for ($i = 0; $i < $n; $i++) {
        for ($j = $i + 1; $j < $n; $j++) {
            if ($arr[$i] === $arr[$j]) {
                return true; // Нашли дубликат
            }
        }
    }
    return false;
}
```

**Визуализация:**
```
Данные:        10        100        1000        10000
Время:         ████       ██████████████       ██████████████████████████████████████████
           (терпимо)      (медленно)            (ОЧЕНЬ медленно!)
              
Если 10 элементов = 0.01 сек
То 100 элементов = 1 сек
А 1000 элементов = 100 сек (1.5 минуты!)
```

## Сравнение всех четырёх

```

│  Сложность │  Название           │  Пример               │
├────────────┼─────────────────────────────────────────────┤
│  O(1)      │  Константная       │  Доступ по индексу    │
│  O(log n)  │  Логарифмическая   │  Бинарный поиск       │

  O(n²)     ████████████████████████████████████████████████████████████
```

## Как определять сложность?

### Правило 1: Один цикл = O(n)
```php
for ($i = 0; $i < $n; $i++) {  // n итераций
    // что-то делаем
}
// Сложность: O(n)
```

### Правило 2: Вложенные циклы = умножаем
```php
for ($i = 0; $i < $n; $i++) {       // n раз
    for ($j = 0; $j < $n; $j++) {    // n раз для КАЖДОГО i
        // что-то делаем
    }
}
// Сложность: O(n × n) = O(n²)
```

### Правило 3: Последовательные блоки = берём максимум
```php
for ($i = 0; $i < $n; $i++) {  // O(n)
    // блок 1
}
for ($j = 0; $j < $n; $j++) {   // O(n)
    // блок 2
}
// Общая сложность: O(n + n) = O(2n) ≈ O(n)
```

### Правило 4: Константы не важны (не пишем)
```php
// 5n операций — это O(n), не O(5n)
for ($i = 0; $i < $n * 5; $i++) { }

// 1000 операций — это O(1), не O(1000)
for ($i = 0; $i < 1000; $i++) { }
```

## Как это помогает на практике?

```
Задача: Написать поиск пользователя

Вариант A: Линейный поиск O(n)
- Просто написать
- Работает для маленьких списков (до 1000)

Вариант B: Бинарный поиск O(log n)
- Сложнее написать
- Требует сортировки
- Незаменим для больших списков

Решение:
- Маленькие данные → Вариант A
- Большие данные → Вариант B
```

## Упражнения

### Легко
Какая сложность у этой функции?
```php
function printAll(array $arr): void
{
    foreach ($arr as $item) {
        echo $item;
    }
}
```

### Средне
Какая сложность у этой функции?
```php
function printPairs(array $arr): void
{
    $n = count($arr);
    for ($i = 0; $i < $n; $i++) {
        for ($j = 0; $j < $n; $j++) {
            echo $arr[$i] . "," . $arr[$j];
        }
    }
}
```

### Сложно
Напиши функцию с сложностью O(n), которая проверяет, есть ли в массиве число больше 100.

---

## Решения

### Легко
**Ответ: O(n)** — один простой цикл.

### Средне
**Ответ: O(n²)** — два вложенных цикла (декартово произведение).

### Сложно
```php
function hasBigNumber(array $arr): bool
{
    foreach ($arr as $num) {
        if ($num > 100) {
            return true; // Нашли — сразу выходим
        }
    }
    return false;
}
// O(n) — в худшем случае пройдём весь массив
// (но если найдём раньше — выйдем ещё быстрее!)
```

## Итоги

- **Big O** показывает, как растёт время при росте данных
- **O(1)** — мгновенно, **O(n)** — линейно, **O(log n)** — очень быстро растёт, **O(n²)** — медленно
- Цель: выбирать алгоритмы с меньшей сложностью
