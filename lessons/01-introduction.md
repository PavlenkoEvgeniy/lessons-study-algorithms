# Урок 1: Что такое алгоритм

## Зачем этот урок

Перед тем как учить алгоритмы, нужно понять, что это такое и зачем они нужны.

---

## Что такое алгоритм?

**Алгоритм** — это пошаговый план для решения задачи.

Алгоритм из жизни: **как заварить чай**

```
Шаг 1: Вскипятить воду
Шаг 2: Положить чай в чашку
Шаг 3: Залить кипятком
Шаг 4: Подождать 3 минуты
Шаг 5: Добавить сахар (по желанию)
```

Каждый шаг понятен. Результат предсказуем. Это алгоритм.

---

## Алгоритм в программировании

Алгоритм — это такая же пошаговая инструкция, но для компьютера.

**Пример:** Найти самое большое число в списке

```
Дано: [3, 7, 2, 9, 4, 1, 8]
Цель: Найти самое большое число
```

**Пошаговый план:**

```
Шаг 1: Берём первый элемент как "текущий максимум"
        Максимум = 3

Шаг 2: Смотрим следующий элемент (7)
        7 больше чем 3? ДА
        Значит максимум теперь = 7

Шаг 3: Смотрим следующий элемент (2)
        2 больше чем 7? НЕТ
        Максимум остаётся = 7

Шаг 4: Смотрим следующий элемент (9)
        9 больше чем 7? ДА
        Максимум теперь = 9

Шаг 5-7: Повторяем для оставшихся элементов
        (2, 4, 1, 8 — ни один не больше 9)

Результат: 9
```

---

## Как записать алгоритм в коде

```php
<?php

/**
 * Найти максимальное число в массиве
 * 
 * Это и есть алгоритм — последовательность шагов
 */

function findMax(array $numbers): int
{
    // Шаг 1: Берём первый элемент как максимум
    $max = $numbers[0];
    
    // Шаг 2: Проходим по всем элементам
    for ($i = 1; $i < count($numbers); $i++) {
        
        // Шаг 3: Если текущий больше максимума — обновляем
        if ($numbers[$i] > $max) {
            $max = $numbers[$i];
        }
    }
    
    // Шаг 4: Возвращаем результат
    return $max;
}

// Проверяем
$numbers = [3, 7, 2, 9, 4, 1, 8];
echo findMax($numbers); // Выведет: 9
```

## Четыре свойства алгоритма

### 1. Конечность
Алгоритм должен завершиться через конечное число шагов.

```
Хорошо:  "Посчитай сумму чисел от 1 до 100" — завершится
Плохо:   "Считай числа бесконечно" — никогда не завершится
```

### 2. Понятность
Каждый шаг должен быть чётким, не двусмысленным.

```
Понятно: "Положи в чашку 1 ложку сахара"
Непонятно: "Положи немного сахара" (сколько это?)
```

### 3. Корректность
Алгоритм должен давать правильный ответ для любых входных данных.

### 4. Эффективность
Алгоритм должен работать быстро и не тратить много памяти.

## Ещё один пример: Поиск друга в списке

**Задача:** Проверить, есть ли друг "Маша" в списке друзей.

```
Друзья: ["Аня", "Борис", "Маша", "Глеб"]
Ищем: "Маша"
```

**Алгоритм — проходим по списку:**

```
Шаг 1: Смотрим "Аня" — это "Маша"? НЕТ
Шаг 2: Смотрим "Борис" — это "Маша"? НЕТ
Шаг 3: Смотрим "Маша" — это "Маша"? ДА! Нашли!
```

```php
<?php

function findFriend(array $friends, string $name): bool
{
    foreach ($friends as $friend) {
        if ($friend === $name) {
            return true; // Нашли!
        }
    }
    return false; // Не нашли
}

$friends = ["Аня", "Борис", "Маша", "Глеб"];
echo findFriend($friends, "Маша") ? "Есть!" : "Нет"; // Есть!
```

## Упражнения

### Легко
Напиши алгоритм (и код), который находит **минимальное** число в массиве:
```php
findMin([5, 2, 8, 1, 9]); // должно вернуть 1
```

### Средне
Напиши алгоритм, который считает **сумму** всех чисел в массиве:
```php
sumArray([1, 2, 3, 4, 5]); // должно вернуть 15
```

### Сложно
Напиши алгоритм, который проверяет, все ли числа в массиве **положительные**:
```php
allPositive([1, 2, -3, 4]); // должно вернуть false
allPositive([1, 2, 3, 4]);   // должно вернуть true
```

---

## Решения

### Легко
```php
function findMin(array $numbers): int
{
    $min = $numbers[0];
    
    foreach ($numbers as $num) {
        if ($num < $min) {
            $min = $num;
        }
    }
    
    return $min;
}
```

### Средне
```php
function sumArray(array $numbers): int
{
    $sum = 0;
    
    foreach ($numbers as $num) {
        $sum = $sum + $num;
    }
    
    return $sum;
}
```

### Сложно
```php
function allPositive(array $numbers): bool
{
    foreach ($numbers as $num) {
        if ($num <= 0) {
            return false;
        }
    }
    return true;
}
```

## Итоги

- **Алгоритм** — это пошаговый план для решения задачи
- Алгоритм должен быть: конечным, понятным, корректным, эффективным
- Любую задачу можно описать как последовательность простых шагов
